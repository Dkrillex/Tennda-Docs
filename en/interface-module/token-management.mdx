<Info>
  API endpoints use the prefix `http(s)://<your-domain>`. Use HTTPS in production to protect authentication tokens; HTTP is recommended for development only.

  A complete management system for user API tokens, supporting create, update, delete, batch operations, and quota & permission control.
</Info>

<Note>
  Note: API keys are tokens you generate under **Personal Center → Access Tokens** for accessing `/api` routes.
</Note>

## User Authentication

The following endpoints require user login authentication. Include `Authorization` and `New-Api-User` in the request headers.

<ParamField header="Authorization" type="string" required>
  User login token, format: `Bearer your_user_token`
</ParamField>

<ParamField header="New-Api-User" type="string" required>
  Current user ID
</ParamField>

---

### List All Tokens

- **HTTP Method**: GET
- **Path**: `/api/token/`
- **Description**: Paginated list of all tokens for the current user

**Query Parameters**

<ParamField query="p" type="integer" default="1">
  Page number
</ParamField>

<ParamField query="size" type="integer" default="20">
  Page size
</ParamField>

**Request Example**

```javascript
const response = await fetch('/api/token/?p=1&size=20', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_user_token',
    'New-Api-User': 'your_user_id'
  }
});
const data = await response.json();
```

**Success Response**

```json
{
  "success": true,
  "message": "",
  "data": {
    "items": [
      {
        "id": 1,
        "name": "API Token",
        "key": "<YOUR_API_KEY>",
        "status": 1,
        "remain_quota": 1000000,
        "unlimited_quota": false,
        "expired_time": 1640995200,
        "created_time": 1640908800,
        "accessed_time": 1640995000
      }
    ],
    "total": 5,
    "page": 1,
    "page_size": 20
  }
}
```

**Error Response**

```json
{
  "success": false,
  "message": "Failed to retrieve token list"
}
```

**Response Fields**

| Field | Type | Description |
|------|------|------|
| `items` | array | List of token objects |
| `total` | number | Total number of tokens |
| `page` | number | Current page number |
| `page_size` | number | Page size |

### Search Tokens

- **HTTP Method**: GET
- **Path**: `/api/token/search`
- **Description**: Search user tokens by keyword and token value

**Query Parameters**

<ParamField query="keyword" type="string">
  Search keyword, matches token name
</ParamField>

<ParamField query="token" type="string">
  Token value search, supports partial matching
</ParamField>

**Request Example**

```javascript
const response = await fetch('/api/token/search?keyword=api&token=sk-123', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_user_token',
    'New-Api-User': 'your_user_id'
  }
});
const data = await response.json();
```

**Success Response**

```json
{
  "success": true,
  "message": "",
  "data": [
    {
      "id": 1,
      "name": "API Token",
      "key": "sk-your-token-placeholder",
      "status": 1,
      "remain_quota": 1000000
    }
  ]
}
```

**Error Response**

```json
{
  "success": false,
  "message": "Failed to search tokens"
}
```

### Create Token

- **HTTP Method**: POST
- **Path**: `/api/token/`
- **Description**: Create a new API token, supports batch creation

**Request Body**

<ParamField body="name" type="string" required>
  Token name, maximum 30 characters
</ParamField>

<ParamField body="expired_time" type="integer">
  Expiration timestamp, -1 means never expires
</ParamField>

<ParamField body="remain_quota" type="integer">
  Remaining quota
</ParamField>

<ParamField body="unlimited_quota" type="boolean">
  Whether quota is unlimited
</ParamField>

<ParamField body="model_limits_enabled" type="boolean">
  Whether model restrictions are enabled
</ParamField>

<ParamField body="model_limits" type="array">
  List of allowed models
</ParamField>

<ParamField body="allow_ips" type="string">
  Allowed IP addresses, comma-separated
</ParamField>

<ParamField body="group" type="string">
  Group name
</ParamField>

**Request Example**

```javascript
const response = await fetch('/api/token/', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_user_token',
    'New-Api-User': 'your_user_id'
  },
  body: JSON.stringify({
    name: "My API Token",
    expired_time: 1640995200,
    remain_quota: 1000000,
    unlimited_quota: false,
    model_limits_enabled: true,
    model_limits: ["gpt-3.5-turbo", "gpt-4"],
    allow_ips: "192.168.1.1,10.0.0.1",
    group: "default"
  })
});
const data = await response.json();
```

**Success Response**

```json
{
  "success": true,
  "message": ""
}
```

**Error Response**

```json
{
  "success": false,
  "message": "Token name is too long"
}
```

### Update Token

- **HTTP Method**: PUT
- **Path**: `/api/token/`
- **Description**: Update token configuration, supports status-only updates and full updates

**Request Body**

<ParamField body="id" type="integer" required>
  Token ID
</ParamField>

<ParamField query="status_only" type="boolean">
  Whether to update status only
</ParamField>

Other fields are the same as the create token endpoint and are all optional.

**Request Example (Full Update)**

```javascript
const response = await fetch('/api/token/', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_user_token',
    'New-Api-User': 'your_user_id'
  },
  body: JSON.stringify({
    id: 123,
    name: "Updated Token",
    expired_time: 1640995200,
    remain_quota: 2000000,
    unlimited_quota: false,
    model_limits_enabled: true,
    model_limits: ["gpt-3.5-turbo", "gpt-4"],
    allow_ips: "192.168.1.1",
    group: "vip"
  })
});
const data = await response.json();
```

**Request Example (Status Only)**

```javascript
const response = await fetch('/api/token/?status_only=true', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_user_token',
    'New-Api-User': 'your_user_id'
  },
  body: JSON.stringify({
    id: 123,
    status: 1
  })
});
const data = await response.json();
```

**Success Response**

```json
{
  "success": true,
  "message": "",
  "data": {
    "id": 123,
    "name": "Updated Token",
    "status": 1
  }
}
```

**Error Response**

```json
{
  "success": false,
  "message": "Token has expired and cannot be enabled. Please update the expiration time or set it to never expire."
}
```

### Batch Delete Tokens

- **HTTP Method**: POST
- **Path**: `/api/token/batch`
- **Description**: Delete multiple tokens in batch

**Request Body**

<ParamField body="ids" type="array" required>
  List of token IDs to delete, required and cannot be empty
</ParamField>

**Request Example**

```javascript
const response = await fetch('/api/token/batch', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_user_token',
    'New-Api-User': 'your_user_id'
  },
  body: JSON.stringify({
    ids: [1, 2, 3, 4, 5]
  })
});
const data = await response.json();
```

**Success Response**

```json
{
  "success": true,
  "message": "",
  "data": 5
}
```

**Error Response**

```json
{
  "success": false,
  "message": "Invalid parameters"
}
```

**Response Fields**

| Field | Type | Description |
|------|------|------|
| `data` | number | Number of tokens successfully deleted |

## Introduction

Poll task status and result by `task_id`. Query every **~5 seconds**; generation usually completes in **30–120 seconds**.

Flow: `queued` → `in_progress` → `completed` / `failed`

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer Token, e.g. `Bearer sk-xxxxxxxxxx`
</ParamField>

## Path parameters

<ParamField path="task_id" type="string" required>
  `id` or `task_id` from the create response
</ParamField>

## Example

```bash
curl -X GET "https://api.gravitex.ai/v1/video/generations/ut-abc123def456" \
  -H "Authorization: Bearer sk-xxxxxxxxxx"
```

## Status

| status | Meaning |
| --- | --- |
| `queued` | Queued |
| `in_progress` | Generating; check `progress` |
| `completed` | Success; read `video_url` or `url` |
| `failed` | See `error.message`; moderation failures: [Get moderation result](/en/api-reference/endpoint/seedance-2.0/get-moderation-result) |
